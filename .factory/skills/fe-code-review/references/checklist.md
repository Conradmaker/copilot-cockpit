# 프론트엔드 코드 리뷰 체크리스트

리뷰 시 4단계 순서로 확인하는 빠른 참조 체크리스트예요.
각 항목의 상세 원칙과 코드 예제는 참조 스킬을 확인하세요.

---

## 1. 코드 컨벤션

→ 상세 원칙과 예제: `fe-code-conventions` 참고

### 가독성

- [ ] 매직 넘버에 의미 있는 이름이 부여되었는가?
- [ ] 복잡한 조건식이 의미 있는 변수로 추출되었는가?
- [ ] 중첩 삼항 연산자 대신 명확한 분기를 사용하는가?
- [ ] 같이 실행되지 않는 코드가 분리되어 있는가?
- [ ] 구현 상세가 적절히 추상화되어 있는가?
- [ ] 시점 이동(위아래 왔다 갔다)이 최소화되었는가?

### 예측 가능성

- [ ] 함수/변수 이름이 동작을 정확히 설명하는가?
- [ ] 이름이 겹치지 않게 관리되는가?
- [ ] 같은 종류의 함수들이 일관된 반환 타입을 가지는가?
- [ ] 숨겨진 사이드 이펙트가 없는가?

### 응집도

- [ ] 함께 수정되는 파일이 같은 디렉토리에 있는가?
- [ ] 매직 넘버가 관련 로직 근처에 상수로 정의되어 있는가?
- [ ] 폼 필드의 유효성 검증과 UI가 함께 위치하는가?
- [ ] 불필요한 추상화 없이, 의미 있는 중복은 허용하는가?

### 결합도

- [ ] 각 함수/컴포넌트가 하나의 책임만 가지는가?
- [ ] Props Drilling이 적절히 해결되었는가? (Composition 우선)
- [ ] 불필요한 의존성이 없는가?

---

## 2. React 패턴

→ 상세 패턴과 예제: `fe-react-patterns` 참고

### 컴포넌트 설계

- [ ] Boolean props 대신 명시적 variant를 사용하는가?
- [ ] 관련 컴포넌트가 Compound Component로 그룹화되었는가?
- [ ] `children`이 render props보다 우선 사용되었는가?
- [ ] 컴포넌트가 적절한 크기로 분리되었는가?

### 상태 관리

- [ ] Context가 구현이 아닌 인터페이스(`state`, `actions`, `meta`)로 설계되었는가?
- [ ] 상태가 적절한 레벨에 위치하는가? (불필요하게 끌어올리지 않음)
- [ ] Provider가 상태 구현의 유일한 소유자인가?

### React 19 (해당 시에만)

- [ ] `forwardRef` 대신 직접 `ref` prop을 사용하는가?
- [ ] `useContext` 대신 `use()`를 사용하는가?

---

## 3. 접근성

→ 상세 규칙과 패턴: `fe-a11y` 참고

### 올바른 구조

- [ ] 인터랙티브 요소가 중첩되지 않았는가? (버튼 안에 버튼 등)
- [ ] 시맨틱 HTML 요소를 우선 사용하는가? (`div onClick` 대신 `button`)
- [ ] 비상호작용 요소에 상호작용 역할(role)을 부여하지 않았는가?

### 의미 전달

- [ ] 모든 인터랙티브 요소에 접근 가능한 이름(label)이 있는가?
- [ ] 같은 이름의 요소에 구분 가능한 추가 설명이 있는가?
- [ ] `aria-*` 속성이 올바르게, 최소한으로 사용되었는가?

### 예측 가능한 인터랙션

- [ ] 버튼 역할과 동작이 일치하는가? (이동 → `<a>`, 동작 → `<button>`)
- [ ] 폼이 `<form>` 태그로 감싸져 있는가?
- [ ] 키보드로 모든 기능을 사용할 수 있는가?

### 시각 정보 보완

- [ ] 이미지에 적절한 `alt` 텍스트가 있는가? (장식용은 `alt=""`)
- [ ] 색상만으로 정보를 구분하지 않는가?

### UI 컴포넌트

- [ ] 모달이 포커스 트래핑, Escape 닫기를 지원하는가?
- [ ] 탭/아코디언이 화살표 키 탐색을 지원하는가?
- [ ] `tabIndex`에 양수 값이 사용되지 않았는가? (0 또는 -1만 허용)

## 4. 성능

→ 상세 최적화 규칙: `fe-react-performance` 참고

### 워터폴 제거 (Critical)

- [ ] 독립적인 비동기 요청이 `Promise.all()`로 병렬 처리되는가?
- [ ] 불필요한 `await`로 블로킹하지 않는가?
- [ ] Suspense 경계가 적절히 설정되었는가?

### 번들 크기 (Critical)

- [ ] barrel 파일 대신 직접 임포트를 사용하는가?
- [ ] 무거운 서드파티 스크립트가 지연 로드되는가?
- [ ] `next/dynamic` 또는 `React.lazy`로 코드 분할을 하는가?
- [ ] 조건부 모듈 로딩을 사용하는가?

### 리렌더 최적화 (Medium)

- [ ] 불필요한 리렌더가 방지되었는가? (`memo`, derived state)
- [ ] 비-프리미티브 기본값이 컴포넌트 외부에 호이스팅되었는가?
- [ ] derived state를 Effect 대신 렌더 시점에 계산하는가?
- [ ] 콜백에서만 사용하는 상태를 구독하지 않는가?

### 서버 사이드 (High)

- [ ] 서버 데이터 캐싱이 적절한가? (`React.cache`, LRU)
- [ ] RSC props에 중복 직렬화가 없는가?
- [ ] 서버 액션에 인증 검사가 있는가?

### 렌더링 (Medium)

- [ ] `content-visibility`가 긴 리스트에 적용되었는가?
- [ ] 정적 JSX가 컴포넌트 외부로 추출되었는가?

- 🔴 **Critical**이 하나라도 있으면 머지하지 않아요
- 🟡 **Warning**은 합의 하에 다음 PR로 미룰 수 있어요
- 🟢 **Suggestion**은 선택적으로 적용해요
